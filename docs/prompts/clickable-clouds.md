# PROMPT для Codex: страница «Нажимаемые облачка»

## 1) Контекст и краткий анализ текущего сайта

Проект — статический сайт на чистом HTML/CSS/JS (без сборки и зависимостей), с общей визуальной базой:
- единый `styles.css` с тёмным градиентным фоном, карточками, rounded-эстетикой;
- верхняя навигация через `<nav>` + ссылки-кнопки;
- mobile-friendly макет через `width:min(1000px,92vw)` и адаптивные сетки;
- локальные стили страниц часто добавляются inline через `<style>` в самом HTML;
- язык интерфейса — русский.

Твоя задача — добавить новую интерактивную страницу, которая визуально вписывается в текущий сайт, но ощущается как «живой» playful-экран.

---

## 2) Цель фичи

Сделать страницу-мини-опыт «Нажимаемые облачка», где пользователь кликает на плавающие облачка-кнопки и получает тактильно-приятный отклик:
- мягкий pulse / micro-bounce;
- маленькие частицы/звёздочки;
- плавная смена оттенка облачка;
- счётчик кликов и streak (серия кликов подряд);
- кнопка «Перемешать облака»;
- корректная поддержка `prefers-reduced-motion`.

---

## 3) Ограничения реализации

ЖЁСТКИЕ ограничения:
- Только чистые HTML/CSS/JS.
- Без внешних библиотек, UI-фреймворков, анимационных libs.
- Без CDN-скриптов.
- Без canvas/WebGL (если не критично). Приоритет — DOM/CSS-анимации.
- Один новый файл страницы: `clickable-clouds.html` (в корне проекта, рядом с `index.html`).
- Добавить ссылку на страницу в основную навигацию сайта.

---

## 4) UX-спецификация

Страница должна включать:
1. Заголовок и короткое описание интеракции.
2. HUD/панель состояния:
   - `Клики: N`
   - `Streak: M`
   - лучший streak за сессию (опционально, но желательно).
3. Кнопка `Перемешать облака`:
   - меняет позиции/параметры облаков;
   - не сбрасывает общий счётчик кликов;
   - streak можно сбрасывать (допустимо).
4. Игровое поле/сцена с 8–14 облаками-кнопками.

Поведение по клику на облако:
- +1 к общему счётчику;
- streak увеличивается, если клики идут с интервалом не более ~1.2 сек (значение можно хранить в константе);
- если интервал больше — streak сбрасывается на 1;
- облако анимируется (pulse + micro-bounce);
- вокруг точки клика вспыхивают 6–10 мини-частиц/звёздочек с затуханием;
- оттенок облачка слегка сдвигается (в мягкой палитре, без резких кислотных цветов).

---

## 5) Визуальный стиль

Стилистически совместить с существующим дизайном:
- использовать токены/переменные из `styles.css` (фон, карточки, бордеры, текст);
- форма облака — мягкая, округлая (border-radius + pseudo-elements/inner glow);
- облака разного размера (например S/M/L), с разными траекториями и скоростью «плавания»;
- обеспечить контраст текста/иконок на фоне облачков;
- hover/focus состояния должны быть заметны и аккуратны.

Предпочтительная палитра облаков:
- холодные светлые оттенки (голубой/сиреневый/мятный),
- лёгкая вариативность hue/saturation/lightness.

---

## 6) Анимации

Нужно реализовать 3 класса анимаций:
1. Пассивное «плавание» облаков:
   - индивидуальная амплитуда, длительность и фаза;
   - разная скорость для глубины сцены.
2. Реакция на клик:
   - короткий scale-up + возврат;
   - микроподпрыгивание (translateY).
3. Частицы/звёздочки:
   - возникают в точке клика;
   - разлетаются на короткую дистанцию;
   - затухают и удаляются из DOM после анимации.

Важно: анимации должны быть лёгкими (transform/opacity), без тяжёлых layout thrashing.

---

## 7) Доступность (A11y)

Обязательно:
- каждое облако — это настоящий `<button>`;
- облака доступны с клавиатуры (Tab/Enter/Space);
- видимый `:focus-visible`;
- счётчики поместить в `aria-live="polite"` (или отдельный live-region) для обновлений;
- не полагаться только на цвет в индикации состояний.

`prefers-reduced-motion`:
- отключить/существенно уменьшить пассивное плавание и burst-частицы;
- оставить функциональность кликов без анимационного шума;
- переходы сделать минимальными.

---

## 8) Адаптивность

Mobile-first:
- комфортная работа одной рукой;
- крупные tap-targets (минимум ~44x44);
- корректное поведение на узких экранах (320px+);
- на десктопе можно увеличить игровую область и плотность облаков.

---

## 9) Структура кода

Внутри `clickable-clouds.html`:
- семантичная разметка (`main`, `section`, `header`, блок сцены);
- локальный `<style>` для специфики страницы + использование общего `styles.css`;
- скрипт внизу страницы:
  - генерация/инициализация облаков;
  - логика клика, streak, best streak;
  - burst-частицы;
  - shuffle;
  - обработка reduced motion.

Код должен быть читаемым, с небольшими комментариями в ключевых местах.

---

## 10) Definition of Done (DoD)

Считать задачу завершённой, когда:
1. Есть рабочая страница `clickable-clouds.html`.
2. Страница доступна из основной навигации сайта.
3. Облака кликаются, анимируются, меняют оттенок.
4. Есть частицы/звёздочки при клике.
5. Работают счётчики `Клики` и `Streak`.
6. Работает кнопка `Перемешать облака`.
7. Учтён `prefers-reduced-motion`.
8. Нет внешних зависимостей.
9. Вёрстка корректна на mobile и desktop.
